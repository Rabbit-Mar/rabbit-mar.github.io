LeetCode 第33题:搜索旋转排序数组

2020年05月16日14时 LeetCode 0  字数统计: 664(字)  阅读时长: 3(分) 评论

33. 搜索旋转排序数组

假设按照升序排序的数组在预先未知的某个点上进行了旋转。
( 例如，数组 [0,1,2,4,5,6,7] 可能变为 [4,5,6,7,0,1,2] )。
搜索一个给定的目标值，如果数组中存在这个目标值，则返回它的索引，否则返回 -1 。
你可以假设数组中不存在重复的元素。
你的算法时间复杂度必须是 O(log n) 级别。
示例 1:
输入: nums = [4,5,6,7,0,1,2], target = 0
输出: 4
示例 2:
输入: nums = [4,5,6,7,0,1,2], target = 3
输出: -1

来源：力扣（LeetCode）

解题分析

当看到有序数组查找值，并且时间要求为O(log n)，优先想到的就是二分查找法，先写个例子

但题目中的数组并不是一个纯排好序的数组，先举个例子

nums 4 5 6 7 8 1 2 3
index 0 1 2 3 4 5 6 7

按照传统的二分查找，第一次的mid=3（ (L+R)/2 ），那么原数组被分为两个数组
数组1

nums 4 5 6 7
index 0 1 2 3

数组2：

nums 8 1 2 3
index 4 5 6 7

可以看到，这个两个数组被拆分成了两种情况，有序数组1，无序数组2.
接下来就可以安装传统二分法处理数组1，但是数组二还是无法处理
但是，我们发现将数组2继续拆分还是会得到两个数组，有序和无序

nums 8 1
index 4 5
nums 2 3
index 6 7

那么思路就有了
如果target在有序数组中就按照传统二分查找做
如果target在无序数组中就继续拆分
